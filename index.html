branch_monitor

여러 GitHub 저장소의 브랜치 / 커밋 / PR 을 한 곳에서 모니터링. 카드 클릭 시 그래프 뷰로 진입.

등록된 저장소

repos.json 을 불러오는 중...
저장소 추가: 우상단 + 저장소 추가 버튼 (또는 직접 repos.json 편집).
각 저장소 조회용 read PAT 는 브라우저 localStorage 에 디바이스 단위 공용 저장 (key: branch_monitor.token.v1), 외부 전송 없음. 새 디바이스에서는 1 회 재입력 필요.
저장소 추가용 write PAT 도 별도 localStorage 저장 (key: branch_monitor.write_token.v1).

저장소 삭제

repos.json 에서 — 항목을 제거하는 commit 을 만듭니다. 실제 GitHub 저장소 자체는 삭제되지 않으며, 카드 목록에서만 사라집니다.
브라우저 localStorage 에 저장된 그 저장소의 read PAT 는 그대로 유지됩니다.

🔑 branch_monitor write PAT

추가 시 사용한 것과 동일한 PAT. 이미 등록되어 있으면 자동 채워집니다.

+ 저장소 추가

새 저장소를 repos.json 에 commit 합니다. 본 commit 은 branch_monitor 저장소 자체에 들어가며, Vercel 이 약 30~60 초 후 자동 재배포하면 카드가 노출됩니다.

🔑 branch_monitor write PAT

branch_monitor 저장소 자체에 commit 하기 위한 PAT. 한 번만 등록하면 이 브라우저에 기억됩니다.
PAT 발급 → (Resource = branch_monitor, Permissions = Contents: Read and write + Metadata: Read-only)
